// Core Systems

Everything You Need to Ship

A full-featured engine that compiles down to a single ES module you can mount anywhere.

Scene System

YAML-based scenes with 7 entity types — camera, mesh, model, lights, and UI. Switch scenes at runtime with a single hook call.

PBR Materials

Full physically-based rendering with albedo, normal, roughness, metalness, and emissive maps. KTX2 GPU-compressed textures supported.

Script System

Unity-like lifecycle with init, update, fixedUpdate, dispose, and collision callbacks. Typed parameters, 60Hz fixed timestep, world-space raycasting, and full input access.

Pluggable Physics

Choose Rapier 3D (WASM) or Crashcat (pure JS). Dynamic, fixed, and kinematic rigid bodies with box, sphere, and capsule colliders. Collision callbacks, sensor/trigger volumes, and full ManaRigidBody API.

Lighting & Shadows

Directional, point, and ambient lights with configurable shadow mapping. Per-entity castShadow and receiveShadow controls.

Prefab System

Reusable entity templates with nested children, visual editor, runtime spawning with auto-initialized physics and scripts, entity destruction, and scene-placed instances.

Shadow DOM Isolation

Your game compiles into a single ES module mounted via Shadow DOM. Styles never leak. Drop it into any page.

Audio System

Sound effects and music via the Web Audio API. Buffer caching, volume control, looping, and automatic browser autoplay handling — all accessible from scripts.

Scripts That Feel Familiar

If you've used Unity or Godot, you'll feel right at home. TypeScript scripts with lifecycle hooks, typed parameters, and full access to Three.js objects.

  • init · update · fixedUpdate · dispose
  • Keyboard, mouse, and axis input
  • Editable params in the inspector
  • Adapter-agnostic rigid body access via ctx.rigidBody
